# Agent OS 구축 체크리스트

이 문서는 구현과 파일럿의 진행 상태를 추적한다. 현행 운영 결정은 `DECISIONS.md`를 따르며, 구축 전 설계 배경은 역사 문서인 `PLAN.md`에서 확인한다.

## 진행 원칙

- 한 번에 하나의 핵심 결정을 확정한다.
- 결정 이유와 폐기한 대안을 기록한다.
- 설계가 확정되기 전에 관련 도구를 설치하지 않는다.
- 기존 설정은 검증된 자산이 아니라 재평가 대상으로 본다.
- 되돌릴 수 있는 작은 구현부터 검증한다.
- 데이터 없이 모델, 토큰 상한, 자동화 방식을 영구 고정하지 않는다.

## 현재 상태

- 현재 단계: `9. 파일럿 준비 완료`
- 구현 상태: 기반 구현 완료
- 파일럿 상태: 기반 T2 Run 1건과 즉시 리뷰 완료, 실제 개발 작업 대기
- 다음 결정: 첫 실제 파일럿 작업

- [x] 작업 모드는 스킬 단계, Worker·Reviewer는 호출형 에이전트, Maintainer는 주기형 워크플로로 구현한다.

### Understanding Gate

- [x] 고영향 설계·복구 불가능 변경·새로운 핵심 개념을 이해 대상으로 정한다.
- [x] 문제·목표·사실·가정·선택지·결정·위험·검증·롤백을 결정 패키지로 확정한다.
- [x] 영구 삭제·되돌릴 수 없는 외부 행동·검증된 복구 경로가 없는 변경에서만 멈춘다.
- [x] 가역적인 저수준 구현과 검증은 자율 실행하고 설명·퀴즈는 마지막에 묶는다.

## 3. 권한·격리·안전

### 실행 프로필

- [x] `Lab`: 격리 환경 내부 풀액세스
- [x] `Project`: 허용된 저장소·볼트 내부 자율 실행, 시스템 권한 질문 없음
- [x] `Production`: 읽기 중심, 외부 행동 명시적 승인

### Full-Access Lab

- [x] Lab의 첫 대상은 최소 시스템 구현 후 실제 작업에서 선정한다.
- [x] Docker Code Lab은 Linux 호환 작업에서 필요성이 생길 때 구축한다.
- [x] macOS System Lab은 닷파일·iOS·macOS 자동화에서 필요성이 생길 때 구축한다.
- [x] 호스트 마운트와 자격증명을 금지하고 공개 인터넷 외 사설 경계를 차단한다.
- [x] 시간, 디스크, 토큰 및 비용 상한을 필수로 둔다.
- [x] patch, 테스트, 실행 기록만 반출하고 Lab을 폐기한다.

### 삭제 및 외부 행동

- [x] 이번 작업에서 만든 임시 파일·빌드 산출물·캐시 삭제를 자동 허용한다.
- [x] 기존 파일·디렉터리, Git 미추적 파일, 영구 데이터의 삭제·초기화는 실행 직전에 확인한다.
- [x] 코드·문서 내부의 일반적인 부분 제거는 파일 삭제 승인에서 제외한다.
- [x] 저장소 루트, 대량 파일, 백업 없는 데이터 삭제는 자동 실행하지 않는다.
- [x] 배포, 메시지, 결제, push 및 PR 생성은 승인받는다.
- [x] 비밀정보는 Lab에 반입하지 않고 출력·기록하지 않는다.
- [x] Project는 Git 복구, Lab은 폐기와 재생성을 기본 롤백으로 사용한다.
